# agent-image：Agent 指导流与能力要点（推荐性）

本文描述 **默认专业路径** 与 **能力告知要点**，供 System Prompt 与产品说明迭代时对照；**不是**与 `*-requirements.md` 中 R-条同等效力的硬性验收条文。Agent **可以偏离**本流，但应理解「为何推荐这条地图」。

权威行为约束（确认闸、完成工具、错误回传、意图内化等）以 **`2026-04-23-agent-image-requirements.md`** 为准。

## 2. 推荐工作流（可偏离）

下列顺序为 **默认心智模型**，非固定状态机：

1. **用户提出需求**
2. **分析** 目标、约束、风格与风险
3. **是否需要追问** —— 澄清模糊点，避免闷头生成
4. **是否需要联网搜索** —— 事实、风格、参考文字信息等
5. **是否需要图像搜索** —— 获取可作 **生图参考输入** 的图像（受工具与版权/玩具范围约束；无专用工具时需降级：用户上传、web-fetch 可得的图等）
6. **生成图像** —— 调用主 / 次生图工具；**须先经 R15 所述 UI 确认（按钮：确认 / 拒绝；粒度：一次工具调用），再触达** 真实 API。**前置：** 若当前会话**未暴露任何生图工具**（用户未配置主生图，见 R3 / R9），则**不得**假装画图、**不得**虚构图像；应通过 **文本回复** 告知用户能力缺失并提示先选定主生图 Model
7. **获取图像结果并检查** —— 在 **所选 LLM 支持多模态** 的前提下，将产出 **以约定形式** 回灌模型，对照需求做对齐与质量判断（具体传图协议见实现规划）
8. **若不对或不满意** —— 尝试其他方案或路径，形成 **改进循环**
9. **在进入下一轮作图或明显更换方向之前** —— **询问用户是否许可**（**软性**规范：主要靠 Prompt 与模型自觉；非 R15 级硬闸）
10. **任意时刻可被用户中断** —— 用户随时可触发 R19 中断；中断后等待中的 R15 确认按「未批准」处理，正在跑的工具按失败处理；Agent 收到回传后应**自然结束本轮**或向用户简短说明，不要试图悄悄继续

## 3. 建议告知 Agent 的综合能力要点

在提示词或产品说明中，可强调 Agent **具备**（在工具与模型能力范围内）：

- **补全上下文** 以提升生图质量 —— 包括但不限于：**更完整的文字描述**、**参考图**、结构化约束（来自搜索或用户补充）。
- **搜索、分析、提问** 等多种手段 —— 在动手改像素之前尽量 **对齐信息与意图**。
- **对生成结果进行检查与评估** —— 对照用户目标做 **自检**（多模态阅读生成图）；发现问题则进入改进循环而非假装完成。
- **在换用其他方向或进入新一轮作图前征得用户许可** —— 减少未经沟通的「擅自开新战役」；与 **追问** 不同，侧重 **大方向/新一轮成本** 上的尊重。

## 4. 实现与产品上的注意点（非验收条款）

- **图像搜索 / 参考图链路** 依赖实际工具与 Provider；未就绪时 Agent 行为应 **降级**，而非虚构图片来源。
- **多模态检查** 依赖 **当前 LLM** 的 vision 能力与 **对话内图像引用** 协议（存储、ID、传模态格式等见需求文档 Deferred to Planning）。
- **「换向前问许可」** 若模型经常遗漏，可通过 **Prompt 强化** 或后续 **轻产品提示** 补救；第一版不强制工程化 enforcement。
